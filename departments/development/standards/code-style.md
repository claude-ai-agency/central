# Code Style Standards

## Allgemein
- TypeScript strict mode immer
- ESLint + Prettier (konfiguriert im Repo)
- Keine console.log in Production Code

## Python
- PEP 8 + Ruff
- Type Hints überall
- Docstrings für alle öffentlichen Funktionen

## Commits
- Immer mit Agent-Prefix: [AGENT-PREFIX]
- Conventional Commits Format
- Keine WIP-Commits auf develop/main

## PRs
- Kein PR ohne Tests
- Kein Merge ohne Review
- DSGVO-relevante Changes: DSGVO Officer taggen
