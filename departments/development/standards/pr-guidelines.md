# PR Guidelines

## Pflicht vor jedem PR
- [ ] Tests grün (lokal + CI)
- [ ] Code Review durch mindestens 1 Peer
- [ ] CODEOWNERS benachrichtigt
- [ ] PR-Template vollständig ausgefüllt

## Review-Zeiten (SLAs)
- P0 Hotfix: Sofort (max 2h)
- P1 Feature: 24h
- P2 Improvement: 48h

## Merge-Regeln
- Squash Merge für Features
- Merge Commit für Releases
- Rebase für Hotfixes
