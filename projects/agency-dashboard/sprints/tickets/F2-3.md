- Playwright `toHaveScreenshot()` für Visual Regression laut SETUP-4
- Test-Szenarien laut US-3 und US-4 Playwright-Test-Anforderungen:

  2. Risiko-Ampel Edge Case: 0% Fortschritt (rot)
  3. Risiko-Ampel Edge Case: 50% Fortschritt (gelb — Grenzwert)
  4. Risiko-Ampel Edge Case: 100% Fortschritt (grün)
  5. Dashboard-Header zeigt roten Alert bei rotem Projekt
  6. Skeleton-Loading-State der Projekt-Übersicht
- Mocked API-Responses via Playwright `route()` mit kontrollierten Daten für jeden Edge Case
- Auth-Mock via `storageState` aus SETUP-4
- Test-Datei: `tests/visual/project-overview.spec.ts`
- Baseline-Screenshots: `tests/screenshots/baseline/project-overview-*.png`
- Abhängigkeit: F2-1 und F2-2 müssen abgeschlossen sein
