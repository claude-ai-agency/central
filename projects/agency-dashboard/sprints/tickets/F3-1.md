## Beschreibung

Server-seitigen Finance-Parser bauen, der `_finance/projects.md` und `_finance/hiring-budget.md` via GitHub REST Raw Content API lädt, GFM-Tabellen mit `unified`/`remark` parsed, und typisierte `BudgetEntry[]`-Objekte zurückgibt. Parser läuft ausschließlich serverseitig — nie im Client-Bundle.

## Technische Details

- GitHub REST Raw Content API laut ADR-001 und ADR-002:
  ```
  GET /repos/{owner}/{repo}/contents/_finance/projects.md
    Accept: application/vnd.github.raw+json
  GET /repos/{owner}/{repo}/contents/_finance/hiring-budget.md
    Accept: application/vnd.github.raw+json
  ```
- Parsing-Pipeline laut ADR-001:
  1. Fetch Raw Markdown via REST
  2. `unified().use(remarkParse).use(remarkGfm)` → AST
  3. Walk AST, finde `table`-Nodes
  4. Extrahiere Zeilen zu `BudgetEntry[]`
- Dependencies laut ADR-001: `unified`, `remark-parse`, `remark-gfm` — Server-side only
- Typdefinitionen in `types/finance.ts`: `BudgetEntry { agentId: string, role: string, budgetMonthly: number, spent: number, currency: 'EUR' }`, `FinanceSummary { totalBudget: number, totalSpent: number, entries: BudgetEntry[] }`
- `lib/finance-parser.ts` — Server-only Marker: `import 'server-only'`
- API Route: `GET /api/finance/budget` → `FinanceSummary`
- React Query Cache: `queryKey: ['finance', 'budget']`, `staleTime: 10 * 60 * 1000` (10 Minuten) laut ADR-002
- Nutzt `lib/github-rest.ts` aus SETUP-3
- Abhängigkeit: SETUP-3 muss abgeschlossen sein
