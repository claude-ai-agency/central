# F1-1: GitHub API — Alle aktiven Agent-Profile aus `_hr/agents/` lesen

**Sprint:** 1
**Owner:** Senior BE Python
**Priority:** P0
**Deadline:** 2026-04-02
**Estimate:** Mittel: 2-4h

## Beschreibung

Next.js API Route bauen, die alle Agent-Profil-Dateien aus dem `_hr/agents/`-Verzeichnis des GitHub-Repos liest, YAML-Frontmatter parsed, und typisierte `AgentProfile[]`-Objekte zurückgibt. Diese Route versorgt die Org-Chart-Komponente (F1-2).

## Technische Details

- GraphQL Query gegen GitHub API laut ADR-002:
  ```graphql
  query OrgChart {
    repository(owner: "owner", name: "central") {
      agentProfiles: object(expression: "main:_hr/agents") {
        ... on Tree {
          entries { name, object { ... on Blob { text } } }
        }
      }
    }
  }
  ```
- YAML-Frontmatter aus `.md`-Dateien parsen: `role`, `agent_id`, `status`, `budget_monthly`, `commit_prefix`
- `gray-matter` oder manueller YAML-Parser für Frontmatter-Extraktion (kein `gray-matter` laut ADR-001 nur für Finance — aber hier sinnvoll)
- Typdefinition `AgentProfile` in `types/agent.ts`: `{ agentId: string, role: string, status: 'active' | 'pending-hire' | 'offline', budgetMonthly: number, commitPrefix: string, reportsTo: string }`
- API Route: `GET /api/agents` → `AgentProfile[]`
- React Query Cache: `queryKey: ['org-chart']`, `staleTime: 5 * 60 * 1000` laut ADR-002
- Nutzt `lib/github-graphql.ts` aus SETUP-3
- Abhängigkeit: SETUP-3 muss abgeschlossen sein
