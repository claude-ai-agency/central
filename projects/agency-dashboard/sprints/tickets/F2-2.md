# F2-2: UI — Projekt-Cards mit Progress, ETA, Risiko-Indikator

**Sprint:** 1
**Owner:** Senior FE React
**Priority:** P1
**Deadline:** 2026-04-07
**Estimate:** Groß: 4-8h

## Beschreibung

React-Komponente `ProjectCard` und die `ProjectOverview`-Seite bauen. Jede Karte zeigt Projektname, Sprint, Fortschrittsbalken, ETA und Risiko-Ampel. Rote Projekte stehen oben. Klick führt zu Detail-View mit Issue-Liste.

## Technische Details

- Client Component mit React Query: `useQuery({ queryKey: ['projects'], ... })` laut ADR-001
- `staleTime: 5 * 60 * 1000` laut ADR-002
- Skeleton-Loader während Loading-State (kein Spinner) laut ADR-002
- Risiko-Ampel als Komponente: `RiskIndicator` mit Props `{ risk: 'green' | 'yellow' | 'red' }`
- Rote Projekte erzeugen visuellen Alert im Dashboard-Header laut US-4 AC-3 (via React Context oder Zustand)
- Sortierung nach Risk (rot oben) kommt bereits von der API (F2-1) — UI zeigt in gelieferter Reihenfolge
- Fortschrittsbalken: `<progress>` oder Tailwind `bg-*`-div, zeigt `progress%`
- ETA-Anzeige: ISO-Datum in lesbarem Format (`dd.MM.yyyy`)
- Klick auf Projekt-Card → `/projects/[id]` Route mit Issue-Liste (US-3 AC-4)
- Detail-Route kann für Sprint 1 minimal sein (einfache Issue-Liste, kein aufwendiges Layout)
- Inline-Error mit Retry bei Fetch-Fehler laut ADR-002
