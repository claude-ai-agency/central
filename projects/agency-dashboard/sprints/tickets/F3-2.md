# F3-2: UI — Token-Kosten-Widget pro Agent (MTD, Week, Today)

**Sprint:** 1
**Owner:** Senior FE React
**Priority:** P1
**Deadline:** 2026-04-06
**Estimate:** Groß: 4-8h

## Beschreibung

Token-Kosten-Widget bauen das Gesamtkosten prominent im Header zeigt und eine Aufschlüsselung pro Agent mit umschaltbarer Tages/Monatsansicht bietet. Budget-Warnbanner bei 80% und 95% Verbrauch integrieren.

## Technische Details

- Client Component mit React Query: `useQuery({ queryKey: ['finance', 'budget'], ... })` laut ADR-001
- `staleTime: 10 * 60 * 1000` (10 Minuten) laut ADR-002
- Skeleton-Loader während Loading-State (kein Spinner) laut ADR-002
- Gesamtkosten im Dashboard-Header: `€XX.XX / €100.00` prominent (US-5 AC-1)
- Tabelle/Liste: pro Agent Name, EUR-Betrag, Token-Anzahl (US-5 AC-2)
- Toggle-Button: "Monat" / "Woche" / "Heute" — schaltet Ansicht um (US-5 AC-3) — Daten kommen aus aggregierten Finance-Daten (F3-1)
- Budget-Warnbanner laut US-6:
  - >80%: gelber Banner im Header mit: aktueller Verbrauch EUR, %, prognostiziertes Monatsende
  - >95%: roter Banner mit Signal Red
  - Banner ist nicht dismissbar (US-6 AC-4)
- Banner-Komponente: `BudgetWarningBanner` mit Props `{ spentPercent: number, spentEur: number, projectedEur: number }`
- Prognose-Berechnung: `(spentEur / verstrichene Tage) * Tage im Monat`
- Inline-Error mit Retry bei Fetch-Fehler laut ADR-002
