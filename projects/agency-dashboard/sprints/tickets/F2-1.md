# F2-1: GitHub API — Projekte aus `projects/` lesen, Status aus brief.md

**Sprint:** 1
**Owner:** Senior BE Python
**Priority:** P1
**Deadline:** 2026-04-05
**Estimate:** Groß: 4-8h

## Beschreibung

Next.js API Route bauen, die alle Projektverzeichnisse aus `projects/` liest, `brief.md` und aktuelle Sprint-Dateien parsed, und angereichert mit GitHub Issues/Milestones eine typisierte `ProjectOverview[]` zurückgibt. Diese Route versorgt die Projekt-Cards (F2-2).

## Technische Details

- GraphQL Query gegen GitHub API laut ADR-002:
  ```graphql
  query ProjectOverview {
    repository(owner: "owner", name: "central") {
      projects: object(expression: "main:projects") {
        ... on Tree {
          entries {
            name
            object {
              ... on Tree {
                entries { name, object { ... on Blob { text } } }
              }
            }
          }
        }
      }
      issues(first: 50, labels: ["agency-dashboard"], states: OPEN, orderBy: {field: UPDATED_AT, direction: DESC}) {
        nodes {
          title, number, state, labels(first: 10) { nodes { name, color } }
          assignees(first: 5) { nodes { login } }
          updatedAt, createdAt
        }
      }
    }
  }
  ```
- `brief.md` parsen: Projektname, Status, ETA aus Markdown-Inhalt extrahieren
- Sprint-Datei (`sprints/sprint-*.md`) parsen: Milestone-Status, Backlog-Items, offene/geschlossene Tickets zählen
- Risiko-Ampel-Berechnung laut US-4 AC-1: `(abgeschlossene Issues / geplante Issues) vs. (verstrichene Sprint-Zeit / Sprint-Dauer)` — >80% on-track = grün, 50-80% = gelb, <50% = rot
- Typdefinition `ProjectOverview` in `types/project.ts`: `{ id: string, name: string, sprint: string, progress: number, eta: string, risk: 'green' | 'yellow' | 'red', issuesClosed: number, issuesTotal: number }`
- API Route: `GET /api/projects` → `ProjectOverview[]`, sortiert nach Risk (rot zuerst) laut US-3 AC-3
- React Query Cache: `queryKey: ['projects']`, `staleTime: 5 * 60 * 1000` laut ADR-002
- Nutzt `lib/github-graphql.ts` aus SETUP-3
- Abhängigkeit: SETUP-3 muss abgeschlossen sein
